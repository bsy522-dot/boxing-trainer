현재 구현 상태 (boxing-trainer-v2.html)
================================================================================

1. 기술 스택
   - Three.js r128 (CDN)
   - 순수 HTML/CSS/JS 단일 파일
   - 브라우저에서 바로 열 수 있음

2. 3D 복서 모델 구조
   - 계층적 관절 구조: root → pelvis → spine → chest → head/arms
   - 각 관절이 THREE.Group으로 구성되어 독립적 회전 가능
   - 부위: 머리, 목, 가슴, 복부, 골반, 양팔(어깨→팔꿈치→글러브), 양다리(힙→무릎→발목)
   - 재질: 피부색, 복싱 글러브(빨강), 반바지(검정), 부츠(검정), 손목 랩

3. 오소독스 가드 포즈 (리서치 기반)
   - 몸통 약 45° 회전 (pelvisRY: -0.45)
   - 앞발 45°, 뒷발 60-70° 각도
   - 뒷발 뒤꿈치 살짝 들림 (rAnkX: -0.15)
   - 무릎 약간 굽힘 (lKnX: 0.2, rKnX: 0.22)
   - 턱 당김 (headRX: -0.12)
   - 뒷손 턱/볼에 밀착 (rElX: -1.8)
   - 앞손 턱에서 약 7인치 앞 (lElX: -1.6, lShZ: 0.6)
   - 팔꿈치 몸에 붙여 바디 보호

4. 구현된 동작들

   [기본 펀치 6종]
   - JAB (1): 앞손 직선 신전, 어깨 턱 보호, 뒷손 턱 유지
   - CROSS (2): 뒷발 피벗 → 힙 회전 → 어깨 회전 → 팔꿈치 피스톤 신전
   - LEAD HOOK (3): 팔꿈치 90° 고정 + 수평 아크 + 앞발 피벗 + 몸통 회전
   - REAR HOOK (4): 뒷손 훅, 뒷발 피벗 + 힙 역회전
   - LEAD UPPERCUT (5): 3/4 스쿼트 → 앞팔 90° 드롭 → 다리 드라이브 수직 상승
   - REAR UPPERCUT (6): 뒷발 피벗 + 힙 회전 + 다리 폭발적 드라이브

   [콤비네이션 5종]
   - 1-2: 잽 → 크로스
   - 1-2-3: 잽 → 크로스 → 훅
   - 1-2-3-6: 잽 → 크로스 → 훅 → 어퍼
   - 1-1-2-3: 더블잽 → 크로스 → 훅
   - 2-3-5-6: 크로스 → 훅 → 어퍼 → 어퍼

   [풋워크 & 디펜스 7종]
   - 전진/후퇴 스텝
   - 좌/우측 이동
   - 슬립 좌/우
   - 밥 앤 위브

5. 인터랙션
   - 마우스 드래그: 360° 카메라 회전 (오빗 컨트롤)
   - 스크롤: 줌 인/아웃
   - 터치 지원 (1핑거 회전, 2핑거 핀치줌)
   - 속도 슬라이더: 0.1x ~ 2.0x
   - 사우스포 미러링 토글
   - 키보드: Space(재생), R(리셋), ←→(스텝)
   - 콤비네이션 시퀀스 표시 UI

6. 애니메이션 시스템
   - 키프레임 기반 보간 (각 동작별 t=0~1 구간)
   - smoothstep 이징 함수
   - null이면 가드 포즈로 자동 복귀
   - 콤보는 순차적으로 각 동작을 재생

================================================================================
현재 한계점 & 개선이 필요한 부분
================================================================================

A. 3D 모델 품질
   - 현재: 기본 도형(실린더, 구체, 박스)으로 구성된 간이 모델
   - 개선: GLTF/GLB 포맷의 리깅된 인체 모델 사용하면 훨씬 사실적
   - 근육 표현, 관절 자연스러운 변형 없음

B. 애니메이션 정확도
   - 현재: 수동으로 설정한 관절 각도 키프레임
   - 개선: 모션 캡처 데이터나 실제 복싱 영상 기반 정밀 키프레임
   - 체중 이동(weight transfer)이 시각적으로 명확하지 않음
   - 발바닥-지면 접촉(IK)이 없어 발이 바닥을 관통할 수 있음

C. 물리적 사실성
   - 키네틱 체인(지면→다리→힙→몸통→어깨→팔→주먹) 순차 전달이
     동시에 일어나는 것처럼 보임 → 순차적 delay 필요
   - 무게중심(Center of Mass) 이동 시각화 없음
   - 지면반력(GRF) 표현 없음

D. 추가 콘텐츠
   - 보디샷 전용 동작 (레벨 체인지)
   - 파링(parry) 동작
   - 피벗(pivot) 풋워크
   - 롤링(rolling with punches) 디펜스
   - 숄더 롤(Philly Shell)
   - 카운터 패턴 (슬립 → 크로스 등)
   - 펀칭 콤보 더 추가 (1-6-3-2 등)

E. UX 개선
   - 동작 중 하이라이트: 현재 움직이는 관절/근육 강조 표시
   - 궤적 표시: 주먹이 지나간 경로를 선으로 표시
   - 정면/측면/위에서 동시에 보는 멀티뷰
   - 타이밍 비트 표시 (각 동작의 리듬)
   - 음성/텍스트 코칭 ("뒷손 턱에서 떼지 마세요!")

아래 프롬프트를 Claude에게 boxing-trainer-v2.html 파일과 함께 전달하세요:

---

이전에 만든 3D 복싱 트레이너 앱(boxing-trainer-v2.html)을 더 완벽하게 개선해줘.

현재 상태:
- Three.js r128 기반 단일 HTML 파일
- 기본 도형으로 만든 관절 인체 모델
- 오소독스 가드 포즈 (바이오메카닉스 리서치 기반)
- 기본 펀치 6종, 콤비네이션 5종, 풋워크 7종
- 키프레임 보간 애니메이션 + smoothstep

개선해야 할 점:

1. 애니메이션 정확도 향상
   - 키네틱 체인 순차 전달: 지면→다리→힙→몸통→어깨→팔→주먹 순서로
     시간차(delay)를 두어 파워가 순차적으로 전달되는 것을 표현해줘
   - 체중 이동을 골반 위치와 무릎 각도 변화로 더 명확히 표현
   - 각 펀치에서 비펀칭 손은 항상 턱 보호 위치를 확실히 유지
   - 잽: 더 빠르고 스냅 있게 (전체 0.4초), 어깨 올려 턱 보호
   - 크로스: 뒷발 힐 아웃 피벗을 명확히, 힙 먼저 회전 후 어깨 따라감
   - 훅: 팔꿈치 90° 고정 상태에서 수평면으로만 회전하도록
   - 어퍼컷: 먼저 아래로 싱크(sink) 후 폭발적 상승

2. 추가 동작
   - 바디 잽 (1B): 레벨 체인지 + 무릎 굽혀 낮은 자세에서 잽
   - 바디 크로스 (2B): 레벨 체인지 + 크로스
   - 피벗 (PIVOT): 앞발 축으로 45-90도 회전
   - 파링 (PARRY): 상대 잽을 뒷손으로 밀어내기
   - 롤 (ROLL): 훅을 U자로 피하며 체중 이동
   - 카운터 콤보: 슬립 오른쪽 → 크로스, 파링 → 크로스 → 훅
   - 추가 콤보: 1-6-3-2, 3-2-3, 1-2-5-2

3. 시각적 개선
   - 주먹 궤적(trail) 표시: 펀치 시 글러브가 지나간 경로를 빨간 선으로 표시
   - 이동하는 관절에 글로우 이펙트
   - 바닥에 발 위치 가이드 원 표시 (스탠스 체크용)
   - 정면/측면 동시 보기 토글 (split view)
   - 각 동작 단계별 정지(step-through) 기능 강화

4. 교육 기능
   - 동작별 핵심 포인트 오버레이 (예: "턱 당기기", "팔꿈치 고정")
   - 잘못된 자세 예시 보여주기 토글 (흔한 실수와 올바른 자세 비교)
   - 초보자/중급자/고급자 난이도별 콤비네이션 분류
   - 각 동작의 속도, 파워, 난이도 표시

5. 3D 모델 개선
   - 관절 부위에 작은 구체(joint indicator) 추가하여 관절 위치 명확히
   - 몸통 근육 윤곽 더 사실적으로 (삼각근, 대흉근 등)
   - 글러브 형태 더 사실적으로 (현재 구체 → 복싱 글러브 형태)

한국어로 전체 인터페이스 유지. 단일 HTML 파일로 만들어줘.
