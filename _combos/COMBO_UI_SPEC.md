# 콤보 학습 모듈 — UI 와이어프레임 스펙 (v1)

대상: `boxing-trainer-v5.html`
연동 데이터: `_combos/combo_data.js` (`CLASSIC_COMBOS`, `COMBO_PUNCH_LABEL`, `COMBO_SCORE_RUBRIC`)
연동 코드: `_combos/integration_snippet.js`

---

## 0. 진입 흐름

```
[훈련 메뉴]
   └─ "콤보 학습" 버튼 (신규) 추가
        ↓
[콤보 선택 화면]   ← CLASSIC_COMBOS 8개 카드
   ├─ 카드: 번호(1-2), 이름, 별점난이도, 스태미나, 한줄설명
   └─ 클릭 시 ↓
[학습 4단계] (단일 콤보 진입)
   1) 시연 (Demo)
   2) 가이드 (Step-by-step)
   3) 메트로놈 (Pace)
   4) 응용 테스트 (Score)
        ↓
[결과 화면]  ← 별점 + XP + 다음 콤보 추천
```

## 2. 학습 4단계

### 공통 헤더

```
┌──────────────────────────────────────┐
│  ←  1-2 잽 크로스    [1/4: 시연]     │
│  ●○○○                                │
└──────────────────────────────────────┘
```

- 진행 인디케이터 4점 (1~4단계 표시)
- 단계명: 시연 / 가이드 / 메트로놈 / 테스트

---

### 단계 1: 시연 (Demo)

```
┌──────────────────────────────────────┐
│                                      │
│       [3D 플레이어 모델]             │
│       자동 콤보 시연                 │
│       BPM 60, 무한 루프              │
│                                      │
│  잽 → 크로스                         │
│  체중: 앞 → 뒤                       │
│                                      │
│  [다시보기] [다음 →]                 │
└──────────────────────────────────────┘
```

- 기존 `playerModel.group`을 사용. 펀치 트리거: combo.timing_ms 따라 setTimeout으로 반복.
- 화면 하단에 진행 중인 펀치 한글명 + 체중 이동 표시.
- 카메라: 정면(player) 시점 (cam preset `1_main`).
- 무한 루프, "다음" 누르면 단계 2로.

---

### 단계 2: 가이드 (Step-by-step)

```
┌──────────────────────────────────────┐
│                                      │
│   STEP 2/7                           │
│                                      │
│   "잽: 왼팔을 턱에서 일직선으로      │
│    뻗어. 손등이 위로, 어깨까지       │
│    같이 회전. 팔꿈치 끝까지 펴되     │
│    잠그지 마."                        │
│                                      │
│   [3D 플레이어 - 잽 자세 동결]       │
│                                      │
│  ←  ●●○○○○○  →                       │
│                                      │
│  [건너뛰기] [다음 →]                 │
└──────────────────────────────────────┘
```

- `combo.coaching_steps` 배열을 한 줄씩 표시.
- 각 스텝마다 3D 모델이 해당 자세로 정지(또는 슬로우 모션 반복).
- 좌우 화살표로 스텝 이동 가능.
- 마지막 스텝 끝나면 `common_mistakes` + `pro_tips` 한 줄씩 카드로 표시 후 "다음".

### 단계 3: 메트로놈 (Pace)

```
┌──────────────────────────────────────┐
│  BPM:  [— 80 +]   [재생/정지]        │
│                                      │
│  ┌──────────────────────────────┐    │
│  │ ● ─── ● ─── ● ─── ● ─── ●    │    │
│  │ Jab   X     Cross           │    │
│  │ 0ms  500ms 1000ms 1500ms    │    │
│  └──────────────────────────────┘    │
│       ↑ 현재 위치 (이동 커서)        │
│                                      │
│  지금 칠 펀치:  [ 잽 ]               │
│  다음 펀치:     [ 크로스 ]           │
│                                      │
│  맞음: 12   놓침: 1                  │
│  [다음 →]                            │
└──────────────────────────────────────┘
```

#### 타이밍 시각화 (각 BPM별)

- 가로축 = 시간(ms), 한 박자 = 60000/BPM ms
- 콤보의 `timing_ms` 배열을 박자 위에 점으로 표시
- 입력 키 매핑:
  - `1` = jab
  - `2` = cross
  - `3` = left_hook
  - `4` = right_hook
  - `5` = left_upper
  - `6` = right_upper
- 박자에 맞춰 입력하면 해당 점이 `green`, 어긋나면 `gray`.

#### BPM별 timing_ms 환산 (예: 1-2 콤보, timing=[0, 380])

| BPM | 한 박(ms) | 콤보 길이 | 박자상 위치 |
|----|----|----|----|
| 60 | 1000 | 380ms | 0박, 0.38박 |
| 80 | 750  | 380ms | 0박, 0.51박 |
| 100| 600  | 380ms | 0박, 0.63박 |
| 120| 500  | 380ms | 0박, 0.76박 |
| 140| 428  | 380ms | 0박, 0.89박 |

→ BPM이 빨라질수록 두 펀치 사이가 거의 한 박이 된다.
→ 학습자는 처음에 BPM 60에서 "탁..탁" 호흡으로, 마지막에 BPM 140에서 "타닥" 한 호흡으로 친다.

#### BPM 컨트롤

- 슬라이더 또는 버튼: 콤보의 `bpm_range`에 클램프
- "+10 BPM" / "-10 BPM" 단축 버튼
- 자동 가속 모드: 30초마다 +10 BPM (옵션)

### 단계 4: 응용 테스트 (Score)

```
┌──────────────────────────────────────┐
│  TEST  -  60초                       │
│  BPM 100, 콤보 1-2 X 5회 입력        │
│                                      │
│  ┌────────────────────────────┐      │
│  │  [3D 헤비백]               │      │
│  │  콤보 카운터: 3/5          │      │
│  │  스코어: 240               │      │
│  └────────────────────────────┘      │
│                                      │
│  지금:  [ 잽 ]                       │
│  다음:  [ 크로스 ]                   │
│                                      │
│  Perfect: 4  Good: 2  Ok: 1  Miss: 1 │
│  남은 시간: 28초                     │
└──────────────────────────────────────┘
```

- 60초 안에 콤보를 5~10회 반복 입력
- 각 펀치 입력 시점이 expected timing 대비 ± 얼마인지 측정
- 결과:
  - Perfect (±80ms 안)
  - Good (±160ms 안)
  - Ok (±280ms 안)
  - Miss (그 밖)
- 콤보 완성 시 콤보 카운터 +1, 보너스 점수 +30
- 잘못된 펀치 입력 시 콤보 카운터 리셋

## 3. 결과 화면 (별점 부여 기준)

```
┌──────────────────────────────────────┐
│                                      │
│       1-2 잽 크로스 완료!            │
│                                      │
│        ★ ★ ★ ★ ☆                    │
│                                      │
│   정확도:  92% (PPGOM)               │
│   타이밍:  85% (avg ±105ms)          │
│   콤보 X:  4/5                       │
│                                      │
│   파워 +2  스피드 +1  XP +84         │
│                                      │
│   [다시] [다음 콤보 →] [메뉴]        │
└──────────────────────────────────────┘
```

### 별점 환산식

```
final_score = accuracy * 0.5 + timing * 0.5
  where
    accuracy = (correct_punches / total_punches)
    timing   = sum(timing_score per punch) / total_punches
       timing_score: perfect=1.0, good=0.7, ok=0.4, miss=0.0

stars =
   final_score >= 0.92 → 5
   final_score >= 0.78 → 4
   final_score >= 0.62 → 3
   final_score >= 0.45 → 2
                       → 1
```

### XP 보상

```
xp = round(final_score * 100 * difficulty)
   ★1 콤보 만점 = 100 XP
   ★3 콤보 만점 = 300 XP
   ★5 콤보 만점 = 500 XP
```

### 스탯 보상 (콤보별 스탯 분배)

| 콤보 | 주 보상 스탯 |
|---|---|
| 1-2, 1-1-2 | speed +1~2 |
| 1-2-3, 2-3-2, 1-2-3-2 | power +1~2 |
| 1-2-5-2, 1-6-3-2 | technique (=defense) +1~2 |
| 6-3-2 | power +2, stamina +1 |

## 4. 인터랙션 / 키 매핑

## 5. 사운드 / 햅틱

- 메트로놈: `playStep()` 또는 신규 `playMetronomeTick()` 박자마다
- 펀치 정확: `playPunch('light')` 또는 `'heavy'` (훅/어퍼는 heavy)
- 미스: 톤다운 sfx 또는 무음
- 콤보 완성: `showComboText('1-2-3 완성!')` (기존 함수 재사용)
- 모바일 햅틱: 정확 시 짧게 50ms, 콤보 완성 시 200ms

## 6. 데이터 저장 (`save.combos[id]`)

```js
save.combos = {
  jab_cross:        { stars: 5, bestScore: 0.94, attempts: 12, lastBpm: 120 },
  jab_cross_hook:   { stars: 3, bestScore: 0.71, attempts: 4,  lastBpm: 80  },
  ...
}
```

- 콤보별 최고 별점 / 최고 점수 / 시도 횟수 / 마지막 BPM 저장
- 잠금 해제 판정에 사용
- 메뉴 카드의 "지난 최고점" 배지에 사용

## 7. 추가 사항 (V2 후보)

- 콤보 빌더: 사용자가 펀치 시퀀스를 만들고 저장
- 멀티 콤보 챌린지: 8개 콤보를 5분 안에 모두 ★3+ 클리어
- 코치 음성 (TTS): coaching_steps 자동 낭독
- 리플레이: 4단계 테스트의 입력 타이밍을 그래프로 복기
